# Gobang（五子棋）项目文件

## 项目概述

- **项目类型**: Cocos Creator 2.4.3 游戏项目
- **游戏类型**: 五子棋（Gobang/Five-in-a-Row）
- **开发语言**: JavaScript（ES5/ES6 混合）
- **游戏引擎**: Cocos2d-html5
- **目标平台**: 移动端网页
- **主要功能**: PvP 对战、PvE 对战（AI）、游戏回放、音效管理、设置管理


### 脚本功能详情

#### 001_GameConfig.js
全局配置对象，包含：
- `BOARD` — 棋盘尺寸、格子大小
- `CHESS` — 棋子类型、颜色
- `GAME_MODE` — 游戏模式（PvP/PvE）
- `AI` — AI 难度配置
- `AUDIO` — 音效设置
- `UI` — 界面配置
- `WIN` — 胜利条件
- `STORAGE_KEYS` — 本地存储键名
- `SCENES` — 场景名称
- `REPLAY` — 回放配置

#### Battle.js
游戏核心逻辑：
- 处理落子和棋盘映射 (`map`)
- 胜负判定算法
- AI 调用和整合
- 悔棋功能
- 回放初始化和控制
- 棋子对象池管理
- UI 更新

#### Chess.js
棋子组件：
- 黑白棋子显示切换
- 落子动画
- 最后落子标记
- 对象池重置

#### AI.js
AI 算法：
- Minimax 算法实现
- 启发式评估
- 威胁分析
- 分批处理支持
- 多难度扩展（easy/medium/hard）

#### AudioManager.js
音效系统：
- 音效加载和缓存
- 背景音乐播放
- 音量控制
- 全局静态 API

#### SettingManager.js
设置管理：
- 音量设置持久化
- AI 难度保存
- localStorage 读写
- 统一 API 接口

#### ReplayManager.js
回放控制：
- 开始/暂停/停止
- 前进/后退跳转
- 播放速度调节
- 快照回放

#### ParticleManager.js
粒子系统管理：
- 程序化创建粒子纹理（Canvas → Texture2D）
- 环境粒子效果播放
- 异步纹理加载处理
- 注意：必须在 onLoad 中延迟初始化，等待纹理加载完成

## 代码风格规范

### 命名约定
- **组件类**: PascalCase（如 `WinPopupManager`）
- **文件名**: PascalCase（如 `Battle.js`）
- **属性**: camelCase（如 `chessWidth`）
- **私有方法**: 下划线前缀（如 `_iswin()`）
- **事件处理**: on 前缀（如 `onRestartClick()`）
- **布尔标志**: is/has 前缀（如 `isPopupShowing`）
- **节点引用**: nd 前缀（如 `ndBase`）
- **精灵帧**: spt 前缀（如 `sptFrames`）

### 组件模式
```javascript
cc.Class({
    extends: cc.Component,
    properties: { /* 属性定义 */ },
    onLoad() { /* 初始化 */ },
    start() { /* 首次激活 */ },
    onDestroy() { /* 清理 */ },
});
```

## 重要说明

### 开发规范与禁忌

#### 绝对禁止的操作
1. **绝对不能重命名已有文件** — 会导致编辑器绑定丢失、脚本组件引用失效
2. **绝对不能批量修改多个文件** — 容易遗漏和出错
3. **绝对不能删除已有文件** — 可能破坏依赖关系

#### 修改前必须确认
1. 修改代码前必须先说明要做什么
2. 说明可能的影响
3. 等用户确认后再执行

#### 预制体修改规范
1. 不能直接修改预制体，必须先确认方案
2. 不能给子节点添加 PrefabInfo（PrefabInfo 只能有一个，指向根节点）
3. 不能修改根节点的组件引用
4. 先读取当前结构，分析现有代码依赖

### 关键技术要点
1. **无模块系统**: 通过字符串名称引用组件
2. **中文注释**: 整个代码库使用中文注释
3. **游戏逻辑**: `count >= 4` 表示五子连珠（count 不包含当前落子）
4. **对象池**: 棋子使用对象池管理，减少 GC 压力
5. **本地存储**: 设置通过 localStorage 持久化

### 性能优化策略
1. **对象池复用**: 棋子节点预创建，回收复用
2. **异步 AI 计算**: 分批处理落子评分，避免主线程阻塞
3. **动画模板复用**: Chess 组件中动画对象只创建一次
4. **标记节点复用**: 最后落子标记节点只创建一次，通过 active 切换

## 粒子系统详细说明

### ParticleManager.js
**路径**: `assets/Script/ParticleManager.js`

**功能概述**:
- 管理游戏中的粒子效果
- 程序化创建粒子纹理（Canvas → Image → Texture2D）
- 支持环境粒子、落子特效、胜利特效三种效果类型

**主要方法**:
| 方法 | 功能 | 说明 |
|------|------|------|
| `playAmbientParticles()` | 播放环境粒子 | 背景漂浮光点效果 |
| `playPlaceChessEffect(position)` | 播放落子特效 | 波纹扩散 + 小粒子爆发 |
| `playWinEffect(position)` | 播放胜利特效 | 庆祝粒子爆发效果 |

**纹理创建流程**:
```javascript
// 1. 创建 Canvas 并绘制
const canvas = document.createElement('canvas');
const ctx = canvas.getContext('2d');
// ...绘制内容...

// 2. 转换为 Image 对象（异步）
const img = new Image();
img.onload = () => {
    // 3. 创建 Texture2D
    const texture = new cc.Texture2D();
    texture.initWithElement(img);
    
    // 4. 创建 SpriteFrame
    const spriteFrame = new cc.SpriteFrame();
    spriteFrame.setTexture(texture, cc.rect(0, 0, 32, 32), false, cc.v2(0, 0), cc.size(32, 32));
    
    // 5. 设置粒子纹理（使用 spriteFrame，不是 texture）
    ps.spriteFrame = spriteFrame;
    
    // 6. 配置粒子属性并播放
    ps.resetSystem();
};
img.src = canvas.toDataURL('image/png');
```

**注意事项**:
1. 粒子纹理必须使用 `ps.spriteFrame`，不是 `ps.texture`
2. Canvas 转纹理需要先转 Image 对象
3. 必须在 Image.onload 回调中处理纹理
4. 不要在组件初始化时就调用粒子播放，等待纹理加载完成
