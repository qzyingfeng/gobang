# Gobang（五子棋）项目文件

## 项目概述

- **项目类型**: Cocos Creator 2.4.3 游戏项目
- **游戏类型**: 五子棋（Gobang/Five-in-a-Row）
- **开发语言**: JavaScript（ES5/ES6 混合）
- **游戏引擎**: Cocos2d-html5
- **目标平台**: 移动端网页
- **主要功能**: PvP 对战、PvE 对战（AI）、游戏回放、音效管理、设置管理

## 目录结构

```
gobang/
├── assets/                      # 主要资源目录
│   ├── Script/                  # JavaScript 游戏逻辑
│   ├── Scene/                   # .fire 场景文件
│   ├── Prefab/                  # 可复用预制体
│   ├── Texture/                 # 图片资源
│   └── resources/               # 其他资源
├── settings/                    # 项目设置
├── project.json                 # 项目配置
├── package.json                 # Node 依赖配置
├── tsconfig.json                # TypeScript 配置
└── jsconfig.json                # JavaScript 配置
```

## 场景文件说明

### 场景列表

| 场景文件 | 路径 | 说明 |
|---------|------|------|
| **Login.fire** | `assets/Scene/` | 主菜单场景，提供游戏入口 |
| **Battle.fire** | `assets/Scene/` | 对战场景，核心游戏界面 |
| **Setting.fire** | `assets/Scene/` | 设置场景，音量和难度配置 |

### 场景功能详情

#### Login.fire（登录/主菜单场景）
- 提供人机对战、人人对战入口
- 进入设置界面的入口
- 初始化游戏设置
- 播放菜单背景音乐

#### Battle.fire（对战场景）
- 包含棋盘和棋子
- 显示对局信息面板
- 控制回放 UI
- 显示胜利弹窗
- 处理游戏核心逻辑

#### Setting.fire（设置场景）
- 音量滑块控制
- AI 难度选择
- 返回/重置按钮
- 保存设置到本地存储

### 场景切换关系
```
Login.fire ──→ Battle.fire
    │              │
    │              ↓
    └──→ Setting.fire
```

### 脚本功能详情

#### 001_GameConfig.js
全局配置对象，包含：
- `BOARD` — 棋盘尺寸、格子大小
- `CHESS` — 棋子类型、颜色
- `GAME_MODE` — 游戏模式（PvP/PvE）
- `AI` — AI 难度配置
- `AUDIO` — 音效设置
- `UI` — 界面配置
- `WIN` — 胜利条件
- `STORAGE_KEYS` — 本地存储键名
- `SCENES` — 场景名称
- `REPLAY` — 回放配置

#### Battle.js
游戏核心逻辑：
- 处理落子和棋盘映射 (`map`)
- 胜负判定算法
- AI 调用和整合
- 悔棋功能
- 回放初始化和控制
- 棋子对象池管理
- UI 更新

#### Chess.js
棋子组件：
- 黑白棋子显示切换
- 落子动画
- 最后落子标记
- 对象池重置

#### AI.js
AI 算法：
- Minimax 算法实现
- 启发式评估
- 威胁分析
- 分批处理支持
- 多难度扩展（easy/medium/hard）

#### AudioManager.js
音效系统：
- 音效加载和缓存
- 背景音乐播放
- 音量控制
- 全局静态 API

#### SettingManager.js
设置管理：
- 音量设置持久化
- AI 难度保存
- localStorage 读写
- 统一 API 接口

#### ReplayManager.js
回放控制：
- 开始/暂停/停止
- 前进/后退跳转
- 播放速度调节
- 快照回放

#### ParticleManager.js
粒子系统管理：
- 程序化创建粒子纹理（Canvas → Texture2D）
- 环境粒子效果播放
- 异步纹理加载处理
- 注意：必须在 onLoad 中延迟初始化，等待纹理加载完成

## 代码风格规范

### 命名约定
- **组件类**: PascalCase（如 `WinPopupManager`）
- **文件名**: PascalCase（如 `Battle.js`）
- **属性**: camelCase（如 `chessWidth`）
- **私有方法**: 下划线前缀（如 `_iswin()`）
- **事件处理**: on 前缀（如 `onRestartClick()`）
- **布尔标志**: is/has 前缀（如 `isPopupShowing`）
- **节点引用**: nd 前缀（如 `ndBase`）
- **精灵帧**: spt 前缀（如 `sptFrames`）

### 组件模式
```javascript
cc.Class({
    extends: cc.Component,
    properties: { /* 属性定义 */ },
    onLoad() { /* 初始化 */ },
    start() { /* 首次激活 */ },
    onDestroy() { /* 清理 */ },
});
```

## 重要说明

1. **无模块系统**: 通过字符串名称引用组件
2. **中文注释**: 整个代码库使用中文注释
3. **游戏逻辑**: `count >= 4` 表示五子连珠
4. **对象池**: 棋子使用对象池管理
5. **本地存储**: 设置通过 localStorage 持久化
